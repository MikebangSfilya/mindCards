# mindCards

## Что за сервис?

Это  сервис для карточек памяти, содержат вопрос/ответ, либо же просто заголовок и его описание, никаких ограничений как конкретно использовать эту вещь не существует. Подходят для запоминания различной информации и интревального повторения.

Карточка состоит из нескольких полей:

1. ID карты, нужна для точного поиска по базе данных, с помощью нее можно вытащить конкретную карту
2. Название карты, может содержать короткую запись/вопрос для конкретной карточки
3. Описание/ответ
4. Время создание карточки: понятно из названия
5. Уровень "изученности" карточки, где 0 это ничего не знаю/только добавил, 5 полностью изучена
6. Флаг "Изучена", если стоит true, то в будущем этой карточке будет отдаваться меньший приоритет

## На данный момент

На данный момент сервис поддерживает стандартный CRUD операции для нескольких пользователей. Для запроса нужно явно указывать в заголовке ключ `X-User-ID` и неотрцитательное чисо больше нуля.

Для того чтобы добавить пользователя нужно отправить запрос на `localhost:8080/user` с методом `POST` с нужной структурой:

```json
{
    "email": "email",
    "password" : "dadadadad"
}
```

Пароль хранится в хэшированном виде, валидация все еще находится на довольно низком уровне и не требует на данный момент четкого следования правилам для Email'ов. Пароль также может быть любым и неограниченным по длине строки(кроме пустой).

Теперь нужно доделать авторизацию с аутенфикацией.

Создание карточки(одной либо нескольких): утсановить заголовок с ID пользователя и передать 3 поля, название, описание и тэг.
`POST /card`

```json
[
{
    "Title": "Go",
    "Description": "Это карта обновленного Сервис слоя",
    "Tag" : "NewModel"
}
]
```

Другие эндпоинты описывать на данный момент не буду.
